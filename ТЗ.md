Тестовое задание заключается в показе и редактировании нескольких форм/блоков одновременно несколькими пользователями с синхронизацией через websocket.
Т.е. если два пользователя находятся на странице и один из них редактирует поле, то значение этого поля меняется и у второго юзера. Юзеров одновременно может быть сколько угодно.

Реализация ожидается через кастомный хук, который бы связывал стейты компонента через сокет. Но в принципе реализация может быть и другой. Главное это спрятать сокеты "под капот", что бы не приходилось дублировать функционал для различных блоков.

Блоков три. В каждом 2-3 поля.
Блоки изначально спрятаны под кнопкой. Т.е. выводятся только кнопки, а по клику на кнопку показывем соответствующий блок. Одновременно могут отображаться несколько блоков. Т.е. кнопки - это не табы. Они независимо включают/выключают отображение соответствующего блока.

Взаимодействие с сервером

Все сообщения отправляемые на сервер или получаемые с сервера имеют формат json.
Сообщения от клиента серверу обязательно имеют поле command - это то, что хочет сделать клиент. Команда. Так же должно быть обязательное поле block - это кода блока, над которым выполняется команда. Остальные поля зависят от команды.
Сообщения от сервера клиенту можно поделить на два вида
1) Сообщения об ошибках в ответ на команду клиента. Они всегда содержат success: false
2) Какие то данные по блокам. У них всегда есть поле block с соответствующим кодом

Для получения данных по блоку, сначала надо на блок подписаться.
Подписка осуществляется json командой {"command": "subscribe", "block": "block1"}
С этого момента, если на сервере произойдут изменения в данных этого блока, сервер отправит новые данные полей всем подписанным на этот блок клиентам.
Сразу после подписки сервер отправляет текущее значение полей блока в виде {"block":"block1","data":{"lname":"Игнатов","fname":"Александр"},"status":{"lname":false,"fname":false}
data - это значения всех полей. status - это состояние блокировки полей (об этом ниже).
Если на сервере изменилось одно или несколько полей, он отправит точно такой же массив как при подписке, но без status. В data будут только изменившиеся поля.
Если какой то юзер кликает на поле (берет в фокус) с целью его отредактировать, нужно заблокировать это поле у всех остальных пользователей. Т.е. у остальных клиентов это поле должно стать readonly. Для этого нужно отправить на сервер команду, блокирующую поле {"command": "focus", "block": "block1", "field": "lname"}
Сервер разошлет всем клиентам (кроме инициатора) уведомление, что поле заблокировано {"block":"block1","focus":"lname"}. Клиенты при получении такого сообщения должны сделать это поле readonly
Когда изначальный клиент снимает фокус с поля, он должен отправить на сервер сообщение, снимающее блокировку {"command": "blur", "block": "block1", "field": "lname"}
Сервер разошлет всем клиентам, кроме инициатора, уведомление о том, что поле разблокировано {"block":"block1","blur":"lname"}

Когда юзер кликает на кнопку с блоком повторно, нужно отписаться от блока командой {"command": "unsubscribe", "block": "block1"}

Таким образом алгоритм взаимодействия с сервером можно описать так
1) При заходе на страницу осуществляем подключение wss://taxivoshod.ru:8999
2) При нажатии на кнопку с блоком - отправляем запрос на подписку и показываем блок с пустыми полями
3) При повторном нажатии на кнопку с блоком - отправляем сообщение на отписку и скрываем блок
4) При получении от сервера сообщения с данными полей - обновляем данные полей в блоке. Если в сообщении есть поле status, проставляем readonly у полей со значениями true
5) Если юзер берет поле в фокус - отправляем сообщение об этом на сервер
6) Если юзер убрал фокус с поля - отправляем об этом сообщение на сервер
7) Если с сервера пришло сообщение о блокировки поля - делаем его readonly
8) Если с сервера пришло сообщение о снятии блокировки с поля - убираем readonly
9) При скрытии блока отправляем сообщение об отписки от блока

Если в любой момент времени соедениние с сервером прерывается, или не получается установить его изначально, закрываем все блоки, и не позволяем на них кликнуть.
Выводим где то сообщение о том, что сервер недоступен.
Каждые 5 секунд повторяем попытку подключения к серверу. Как только это удалось, кнопки блоков разблокируем, надпись об ошибке убираем.
